# DEX/DeFi 跨链质押学习记录 - 2026-05-27

## 学习目标
了解去中心化交易所（DEX）、质押（Staking）、跨链交易等核心区块链技术及其常用开发框架，为以后构建自己的 DeFi 应用做准备。

## 一、核心概念与技术要点

### 1. 去中心化交易所（DEX）
- **自动做市商（AMM）**：通过常量乘积公式（x*y=k）或其变体（如 Curve 的稳定币公式、Uniswap V3 的集中流动性）实现链上交易。
- **关键组件**：
  - **工厂（Factory）**：用于创建交易对。
  - **交易对（Pair / Pool）**：存储两种资产的储备，处理 swap、mint、burn 等操作。
  - **路由器（Router）**：提供用户友好的 swap 接口（如计算最佳路径、滑点保护）。
  - **费用**：交易手续费分配给流动性提供者（LP）或协议 treasury。
- **代表实现**：
  - Uniswap V2（x*y=k）、V3（集中流动性 + 费用分层）
  - SushiSwap（Uniswap 分支，额外治理代币 SUSHI）
  - PancakeSwap（在 BSC 上的 Uniswap 克隆）
  - Curve Finance（专注稳定币低滑点交易）
  - Balancer（多资产、可自定义权重的池子）

### 2. 质押（Staking）
- **单币质押**：用户锁定单一资产获得奖励（往往是协议治理代币）。
- **LP 质押**：将 DEX LP 票据质押以获得额外奖励（挖矿）。
- **质押池合约模式**：
  - **奖励分配**：按时间或区块增发，常见模型有 `rewardPerToken()`、`notifyRewardAmount()`。
  - **撤回与惩罚**：有些质押包含提前解锁罚款（如 Curve 的 veModel 锁定）。
- **代表合约**：
  - Aave 的安全模块（Safety Module）质押 AAVE 获得安全奖励。
  - Synthetix 质押 SNX 发行合成资产。
  - LDO 质押（Lido）获取 stETH 及治理权。
  - 各种农场（Farm）合约：如 MasterChef（SushiSwap）用于 LP 质押奖励。

### 3. 跨链交易（Cross‑Chain）
- **锁定与铸造（Lock‑and‑Mint）**：在源链锁定资产，目标链发行对应的锚定资产（如 WBTC、renBTC）。
- **验证人（Validators） / 中继（Relay）**：监听源链事件，在目标链执行铸造或释放。
- **常见框架**：
  - **Chainlink CCIP**（跨链互操作性协议）：提供安全的消息传递和代币转移。
  - **CCTP（Circle Cross‑Chain Transfer Protocol）**：USDC 原生跨链转移，由 Circle 官方实现。
  - **Hop Protocol**：基于 bonder 和 AMM 的乐观跨链桥。
  - **Connext**：基于状态通道的快速跨链转移。
  - **Polygon POS Bridge**、**Arbitrum Bridge**、**Optimism Bridge**：L2 到 L1 的官方桥。

## 三、实践建议（步骤化）

1. **环境准备**
   - 安装 Node.js (≥18)、Git。
   - 选择框架：推荐 **Foundry**（高速）或 **Hardhat**（插件丰富）。
   - 初始化项目：`forge init my-defi` 或 `npx hardhat`.

2. **学习基础合约**
   - 阅读并跑通 Uniswap V2 Pair 基本 swap、mint、burn 测试（OpenZeppelin Test Helpers）。
   - 实现一个最小的 AMM（x*y=k）以理解核心逻辑。
   - 查看 Uniswap V3 Pool 的 `swap`、`mint`、`burn`，研究 tick、sqrtPriceX96、费用计算。

3. **前端对接**
   - 安装 `ethers` 或 `viem` + `wagmi`。
   - 使用 `useAccount`、`useProvider`、`useContractRead`/`useContractWrite` 读取池储备。
   - 调用 `router.swapExactTokensForTokens` 进行交换，处理 slippage 与 deadline。

4. **质押合约实验**
   - 部署一个简单的 Staking ERC20 奖励池（参考 `ZeframLou/playpen`）。
   - 测试 `deposit`、`withdraw`、`getReward`，观察奖励随时间线性增长。
   - 添加奖励铸造函数（仅用于测试网，主net需谨慎）。

5. **跨链原型**
   - 在两条本地链（如 Hardhat 两个实例）上部署锁定合约与铸造合约。
   - 使用脚本监听 `Lock` 事件，触发目标链的 `mint`。
   - 研究 CCTP 示例代码，了解 USDC 的跨链流程。

6. **安全审计**
   - 使用 `slither .` 检测重入、未授权访问、算术溢出等。
   - 在测试网上跑一下 `forge test -vv` 确保所有单元测试通过。
   - 若准备上主net，考虑召唤第三方审计或使用正式的 bug bounty 平台。

7. **监控与治理**
   - 为核心合约添加 `Pauseable`（OpenZeppelin）以便紧急情况下暂停。
   - 实现简单的治理投票（如 `GovernorAlpha`）或直接使用已有 DAO 框架（如 Aragon、Snapshot）。
